MATLAB and Simulink Files for the Proposed Method
Overview
This repository contains five MATLAB-related files that support the implementation and validation of the proposed method presented in the paper.
File List
test.m — Prototype implementation of the proposed algorithm
hafl.m — Auxiliary function used by test.m
restore.m — Auxiliary function used by test.m
model1.slx — Simulink model for Model 1
model2.slx — Simulink model for Model 2

Description of Each File
1. test.m
test.m is the main MATLAB script and provides a prototype implementation of the proposed algorithm.
Reviewers and readers may use this file to understand the basic iterative procedure and implementation framework of the method. It should be noted that the example implemented in test.m is a simplified version of Model 2. Therefore, the purpose of this script is to illustrate the core concepts and algorithmic workflow rather than to directly reproduce all numerical results reported in the paper.
2. hafl.m
hafl.m is an auxiliary function called by test.m. It supports the corresponding computations required during the execution of the main algorithm.
3. restore.m
restore.m is another auxiliary function used by test.m. It performs supporting calculations required by the main implementation.
4. model1.slx
model1.slx is the Simulink model corresponding to Model 1 presented in the paper.
The model contains:
System data
Control law
Disturbance settings
Simulation configuration
The output responses under both uncontrolled and controlled conditions can be observed directly through the Scope block.
The control input is generated by multiplying the multi-segment delayed outputs by the corresponding control law.
5. model2.slx
model2.slx is the Simulink model corresponding to Model 2 presented in the paper.
The file contains:
System data
Control law
Observation settings
Disturbance information
Other simulation parameters required for reproducing the reported results
Detailed information regarding the model matrices, control law, observation period, and the number of delayed outputs can be found in the manuscript.

Important Notes
Relationship Between test.m and the Simulink Models
The control laws used in model1.slx and model2.slx cannot be obtained directly by simply modifying the matrix information contained in test.m.
The implementation provided in test.m is intended as a preliminary and illustrative version of the proposed algorithm. While it captures the essential ideas and iterative structure, it does not include the complete computational procedure required to generate the exact control laws used in Models 1 and 2.
Consequently:
test.m should be viewed as a demonstration of the algorithmic framework.
model1.slx and model2.slx should be used to verify the correctness and effectiveness of the results reported in the paper.
Reproducibility
To understand the proposed method:
1.Run test.m and examine the iterative procedure.
2.Review the auxiliary functions hafl.m and restore.m.
3.Open model1.slx and model2.slx in Simulink.
4.Compare the simulation responses with the results reported in the manuscript.
This combination of MATLAB and Simulink files provides both:
A clear illustration of the algorithmic principles.
A verification platform for the numerical examples presented in the paper.
